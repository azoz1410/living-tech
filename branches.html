فهم الـ Branches في Git

المجلس التقني

📋 ما هو الـ Branch؟

الـ Branch (الفرع) هو خط تطوير منفصل يسمح لك بالعمل على ميزات مختلفة دون التأثير على الكود الرئيسي. تخيل الفرع كطريق جانبي يسمح لك بالتجريب والتطوير بحرية.

🔧 مستكشف أوامر Git التفاعلي

اكتشف أوامر Git وتعلم وظائفها:

git ▼
branch إنشاء، عرض، أو حذف الفروع
checkout التبديل بين الفروع أو استعادة الملفات
switch التبديل بين الفروع (أمر حديث)
merge دمج فرع في الفرع الحالي
push رفع التغييرات إلى المستودع البعيد
pull جلب ودمج التغييرات من المستودع البعيد
add إضافة الملفات إلى منطقة التحضير
commit حفظ التغييرات مع رسالة وصفية
status عرض حالة الملفات في المستودع
log عرض تاريخ الالتزامات
clone نسخ مستودع من مكان بعيد
remote إدارة المستودعات البعيدة

🏢 مثال عملي: شركة تطوير التطبيقات

🚀 شركة "تطبيقاتي"

شركة ناشئة تطور تطبيق للتجارة الإلكترونية. الفريق مكون من:

👨‍💻 أحمد - مطور Frontend
👩‍💻 فاطمة - مطورة Backend
👨‍💼 خالد - مدير المشروع

📅 سيناريو العمل اليومي

main
🌅 بداية اليوم

الفرع الرئيسي (main): يحتوي على الكود المستقر والجاهز للإنتاج

التطبيق الأساسي: صفحة المنتجات + نظام الدفع البسيط
feature
🛒 أحمد يطور ميزة السلة

فرع جديد: feature-shopping-cart

1 ينشئ فرع جديد من main
2 يطور واجهة السلة
3 يختبر الميزة بشكل منفصل
$ git checkout -b feature-shopping-cart
$ git add cart-component.js
$ git commit -m "Add shopping cart UI"
feature
💳 فاطمة تطور نظام الدفع

فرع موازي: feature-payment-gateway

✨ تعمل بشكل مستقل عن أحمد
🛡️ لا تؤثر على عمل الآخرين
🚀 يمكن تطويرها بسرعة
$ git checkout -b feature-payment-gateway
$ git add PayPal-API.js
$ git commit -m "Integrate PayPal API"
hotfix
🚨 مشكلة عاجلة في الإنتاج!

فرع الإصلاح العاجل: hotfix-login-bug

⚠️ المستخدمون لا يستطيعون تسجيل الدخول!
🔧 خالد ينشئ فرع إصلاح عاجل من main
✅ يصلح المشكلة ويدمجها فوراً
$ git checkout -b hotfix-login-bug
$ git add hotfix-login.js
$ git commit -m "Fix login validation error"
$ git checkout main && git merge hotfix-login-bug

🎮 الملعب التفاعلي للأوامر

جرب أوامر Git للفروع

اكتب الأوامر أدناه وشاهد النتيجة:

$
مرحباً! جرب كتابة أوامر Git هنا... أوامر مفيدة للتجربة: • git branch - عرض جميع الفروع • git branch feature-new - إنشاء فرع جديد • git checkout feature-new - الانتقال للفرع • git checkout -b feature-test - إنشاء والانتقال للفرع • git merge feature-new - دمج الفرع • git branch -d feature-new - حذف الفرع

🎯 سيناريوهات عملية

🆕 إضافة ميزة جديدة

تريد إضافة ميزة تسجيل دخول جديدة دون التأثير على الكود الرئيسي.

git checkout -b feature-login git add login.html git commit -m "Add login page" git checkout main git merge feature-login
🚨 إصلاح عاجل

اكتشفت خطأ في الإنتاج ويجب إصلاحه فوراً.

git checkout main git checkout -b hotfix-critical # إصلاح الخطأ git add . git commit -m "Fix critical bug" git checkout main git merge hotfix-critical
🔬 تجريب فكرة

لديك فكرة جديدة وتريد تجربتها دون المخاطرة بالكود الموجود.

git checkout -b experiment-ui # جرب تغييرات جذرية git add . git commit -m "Experimental UI changes" # إذا لم تعجبك النتيجة git checkout main git branch -D experiment-ui

📊 حالات الفروع

✅ فرع محدث (Up to date)

فرعك يحتوي على جميع التغييرات من الفرع الرئيسي.

⬆️ فرع متقدم (Ahead)

فرعك يحتوي على commits جديدة لم يتم دمجها بعد في الفرع الرئيسي.

⬇️ فرع متأخر (Behind)

الفرع الرئيسي يحتوي على تغييرات جديدة لم تصل إلى فرعك بعد.

🧠 اختبر فهمك

السؤال الأول: ما هو الأمر الصحيح لإنشاء فرع جديد والانتقال إليه مباشرة؟

git branch new-feature
git checkout -b new-feature
git create new-feature
git switch new-feature

السؤال الثاني: ما هي الفائدة الرئيسية من استخدام الفروع في Git؟

تسريع عملية التطوير فقط
العمل على ميزات مختلفة بشكل منفصل دون التأثير على الكود الرئيسي
توفير مساحة تخزين على الحاسوب
تقليل عدد الملفات في المشروع

السؤال الثالث: ما هو الأمر الصحيح لحذف فرع محلي؟

git remove branch-name
git branch -d branch-name
git delete branch-name
git branch --remove branch-name

💡 نصائح:

🎯 أسماء الفروع الواضحة

استخدم أسماء وصفية مثل feature-login أو bugfix-payment بدلاً من my-branch.

🧹 تنظيف الفروع

احذف الفروع المدموجة بانتظام باستخدام git branch -d branch-name للحفاظ على نظافة المشروع.

🔄 التحديث المستمر

اسحب التغييرات من main بانتظام إلى فرعك باستخدام git merge main أو git rebase main.

🎨 استراتيجية الفروع

اتفق مع فريقك على استراتيجية واضحة: متى تنشئ فرعاً؟ كيف تسميه؟ متى تدمجه؟
